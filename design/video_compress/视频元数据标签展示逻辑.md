# 📄 产品需求文档 (PRD) 补遗：视频元数据标签展示逻辑

## 1. 功能概述

在“视频压缩详情页”中，由于纯文件名缺乏视觉辨识度，系统需要读取本地视频文件的底层元数据，并根据特定规则在视频预览卡片（Item）上智能化地展示属性标签（Tag）。该功能旨在帮助用户快速识别“空间杀手”和“虚胖视频”，提供强有力的压缩和清理直觉。

## 2. 标签分类定义

系统底层提取的视频元数据标签严格划分为以下四类：

1. **空间维度（Resolution 分辨率）**：描述画面的物理像素尺寸。
2. **时间维度（Frame Rate 帧率）**：描述视频每秒包含的静态画面张数 (FPS)。
3. **压缩维度（Codec 编码格式）**：描述视频文件所使用的数学压缩算法类型。
4. **数据流维度（Bitrate 码率）**：描述视频播放时平均每秒消耗的物理数据流体积。

---

## 3. 标签触发与 UI 展示规则 (核心需求)

系统遍历本地视频列表时，需运行以下过滤逻辑。为了保持界面极简干净，**不符合以下任何触发条件的常规视频（如标准1080P/30帧/H.265编码），一律不展示任何标签。**

| 标签大类 | 标签文本 (UI Show) | 触发技术条件 (底层检测) | 视觉权重 | 产品目的 / 引导文案 |
| --- | --- | --- | --- | --- |
| **空间维度** | `4K UHD` | 视频垂直分辨率 $\ge 2160$ px | **极高** | 告知用户这是超清大文件，降采样能瞬间释放巨量空间。 |
| **空间维度** | `8K` | 视频垂直分辨率 $\ge 4320$ px | **最高** | 顶级空间杀手，强烈建议压缩。 |
| **时间维度** | `60 FPS` | 视频底层帧率 $55 \le \text{FPS} \le 65$ | 中等 | 提示高帧率录屏，限制到 30 帧可直接砍掉一半多余图片数据。 |
| **时间维度** | `Slo-mo` | 视频底层帧率 $\ge 120$ | **高** | 识别系统级“慢动作”特异录制，防止用户误触遗忘导致空间堆积。 |
| **压缩维度** | `H.264` | 视频编码格式为 `avc1/H.264` 且文件体积 $> 200\text{MB}$ | 中等 | 暗示用户这是老旧编码，重新转码为 H.265 可直接瘦身 50%。 |
| **压缩维度** | `ProRes` | 检测到 Apple ProRes 专业级未压缩格式 | **极高** | 针对专业无损原片高亮警告（1分钟可达数GB），普通用户必压。 |
| **数据流维度** | `High Rate` | 1080P视频码率 $\ge 30\text{Mbps}$ 或 4K视频码率 $\ge 60\text{Mbps}$ | **高** | 标记“虚胖视频”，点明其由于码率溢出造成的空间浪费。 |

## 4. UI 渲染与冲突消解逻辑

1. **单行数量上限**：为了防止界面臃肿，单个视频卡片（Item）上**最多只能同时并存展示 2 个标签**。
2. **多标签冲突降序**：若单个视频同时触发了 3 个或以上的标签（例如：一个既是 4K、又是 60fps、还是 H.264 的高码率视频），系统必须按照以下**权重降序**保留前 2 个标签进行渲染，其余忽略：

$$\text{视觉权重：空间维度 (分辨率) } > \text{ 时间维度 (帧率) } > \text{ 压缩/数据流维度 (编码/码率)}$$


* *示例*：同时触发 `4K UHD`、`60 FPS`、`High Rate`，UI 最终仅提炼展示 `[4K UHD]` 和 `[60 FPS]`。


3. **视觉样式规范**：
* 标签统一采用细圆角矩形胶囊舱（Pill Badge）样式。
* 背景色采用与整体白色主题呼应的冷灰色（如 `#f6f8fa`），边框 `#e1e4e8`，文字颜色 `#57606a`，确保在预览列表里既清晰，又不抢整体视觉焦点。
* 高危标签（如 `8K` 或 `ProRes`）允许开发团队配置微弱的淡橙色或淡红色半透明背景作为警告。

## 5. 前端开发逻辑参考（伪代码）

```javascript
function getRenderTags(videoAsset) {
    let activeTags = [];

    // 1. 分辨率边界检测
    if (videoAsset.height >= 2160 && videoAsset.height < 4320) activeTags.push("4K UHD");
    else if (videoAsset.height >= 4320) activeTags.push("8K");

    // 2. 帧率边界检测
    if (videoAsset.fps >= 55 && videoAsset.fps <= 65) activeTags.push("60 FPS");
    else if (videoAsset.fps >= 120) activeTags.push("Slo-mo");

    // 3. 编码边界检测
    if (videoAsset.codec === "H.264" && videoAsset.fileSizeMB > 200) activeTags.push("H.264");
    else if (videoAsset.codec === "ProRes") activeTags.push("ProRes");

    // 4. 码率边界检测
    if (videoAsset.bitrateMbps >= 40) activeTags.push("High Rate");

    // 5. 裁剪策略：依照 PRD 权重降序排序，最多截取前 2 个
    return activeTags.slice(0, 2);
}

```
